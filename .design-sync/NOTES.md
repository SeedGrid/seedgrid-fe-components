# design-sync notes — @seedgrid/fe-components

## Shape & build
- Package shape (no Storybook). `apps/showcase` is a Next.js demo app, not a story harness.
- pnpm monorepo (root `seedgrid-fe`, pins `pnpm@9.12.0`; system pnpm 10.x present). If corepack fights the pinned version, set `COREPACK_ENABLE_STRICT=0`.
- DS package: `packages/seedgrid-fe-components`, built entry `dist/index.js`, types `dist/index.d.ts`. Build: `pnpm -F "@seedgrid/fe-components..." build` (the `...` pulls workspace deps fe-core/fe-commons/fe-theme).
- Converter `--node-modules` → `packages/seedgrid-fe-components/node_modules` (has its own react/react-dom).

## Styling (CRITICAL — two layers, both required)
- **This DS styles via Tailwind utility classes, NOT pure CSS-in-JS.** Components emit Tailwind class names (`inline-flex`, `font-medium`, `rounded-full`, and arbitrary values like `bg-[var(--sg-btn-bg)]`). The package ships NO compiled CSS — the consuming app generates it. Without the Tailwind stylesheet, every component renders as unstyled default HTML (confirmed: buttons were colorless gray until the CSS was added).
- **Layer 1 — Tailwind stylesheet (`cfg.cssEntry`)**: generated by compiling the showcase's `tailwind.config.ts` + `src/app/globals.css`, scanning the component source. Output goes to `packages/seedgrid-fe-components/.ds-assets/seedgrid-tailwind.css` (cssEntry must live INSIDE the package dir; gitignored). This is folded into `cfg.buildCmd` so re-sync regenerates it before the converter runs. Provides the utility classes + Preflight reset + base `:root` shadcn tokens (light theme).
- **Layer 2 — `SeedThemeProvider`** (export of `@seedgrid/fe-theme`, wired as `cfg.provider`, with `@seedgrid/fe-theme` in `cfg.extraEntries`): injects the runtime theme tokens (`--primary`, `--secondary`, the rich `--sg-{palette}-{50..900}` scale, `--radius`, etc.) onto `document.documentElement`. Designs built with the DS must wrap in `SeedThemeProvider` to be themed.
- Token vocabulary: shadcn-style HSL triples (`--primary`, `--secondary`, `--accent`, `--destructive`, `--background`, `--foreground`, `--border`, `--ring`, `--radius`) + the SeedGrid `rgb`-triple palette (`--sg-primary-500`, `--sg-secondary-500`, `--sg-tertiary-500`, `--sg-warning-500`, `--sg-error-500`, …) + per-component `--sg-btn-*`, `--sg-badge-*` tokens set inline by the component from severity.
- **Component severity colors differ by component.** SgButton maps severities to the rich `--sg-*` palette → all 7 severities vivid/distinct. SgBadge maps success→`--tertiary`, warning→`--accent`, info→`--secondary` (shadcn neutrals) → in the DEFAULT theme success/warning/info render muted gray; only primary (green) and danger (red) are vivid. Authentic, not a bug.
- Additional providers checked: `SgComponentsI18nProvider` (pt-BR i18n) and `SgEnvironmentProvider` were NOT needed for the solo set (sensible defaults). Add via `cfg.provider.inner` only if a component errors on missing context.
- Previews use INLINE styles for layout glue (no Tailwind in preview source), since the bundle only ships the component-scoped Tailwind utilities.

## Re-sync risks
- **Tailwind CSS regeneration is load-bearing.** `cfg.buildCmd` regenerates `.ds-assets/seedgrid-tailwind.css` by invoking the showcase's tailwind CLI. If `apps/showcase` is removed, its `tailwind.config.ts`/`globals.css` change, or its `tailwindcss` devDep is uninstalled, the CSS won't regenerate and the bundle silently desyncs to unstyled. The generated file is gitignored — a fresh clone MUST run `cfg.buildCmd` (not a bare converter run) before the converter.
- cssEntry path is bounded to the package dir; if the package is relocated the path breaks.
- Provider = `SeedThemeProvider` default theme only. If the brand theme is customized (non-default seed), previews show the default green theme, not the brand's.
- No `[FONT_MISSING]`/`[FONT_*]` warnings on first build — DS uses system fonts. If a brand font is later added, validate will flag it.
- Docs are JSON (`dist/ai/*.manifest.json`), not `.md` — `.prompt.md` is synthesized. If richer prompt docs are wanted later, generate `.md` from the manifest into a docsDir.

## Re-sync risks (updated)
- `cfg.componentSrcMap` now pins the 50-card scope. If the package adds a new component you DO want carded, it won't appear until you author its preview (it's not auto-excluded, but unauthored = floor card; with the curated approach floor cards weren't wanted). If the package RENAMES one of the 33 pruned exports, the stale `componentSrcMap` null silently stops matching → it reappears as a floor card; reconcile the map on major version bumps.
- Tailwind CSS regeneration via `cfg.buildCmd` is still load-bearing (see above). Note: tailwind v3.4 skips rewriting `seedgrid-tailwind.css` when content is byte-identical (mtime unchanged) — that's normal, not a stale build.
